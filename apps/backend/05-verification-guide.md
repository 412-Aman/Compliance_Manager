# Backend Verification Guide

To ensure the restored backend is fully functional, follow these steps to test the core features: AI scoring, regulatory polling, and live ingestion.

## 1. Environment & Dependency Setup
First, install all necessary dependencies:
```bash
cd apps/backend
pip3 install -r requirements.txt
```

Create a `.env` file in `apps/backend/` if it doesn't exist:
```env
DATABASE_URL=sqlite+aiosqlite:///./compliance.db
OPENAI_API_KEY=your_openai_api_key_here
POLYGON_RPC_URL=https://polygon-rpc.com
```

## 2. Start the API Server
Run the FastAPI application:
```bash
cd apps/backend
python3 -m uvicorn src.main:app --reload
```
Open **[http://localhost:8000/docs](http://localhost:8000/docs)** in your browser.

## 3. Test Regulatory Intelligence
1. Find the `POST /api/v1/regulatory/poll` endpoint.
2. Click **Try it out** -> **Execute**.
3. This will trigger the real scraper to fetch news from FATF/FCA.
4. Check **`GET /api/v1/regulatory/updates`** to see the analyzed results (summaries and obligations extracted by AI).

## 4. Test AI Scoring & Alerts
1. **Create a Customer**: 
   - `POST /api/v1/customers`
   - Payload: `{"label": "Etherscan Hot Wallet", "chain": "Etherium", "address": "0x396343362be2A4dA1cE0C1C210945346fb82Aa49"}`
2. **Simulate a Transaction**:
   - Open a **new terminal** and run the ingestion worker:
     ```bash
     cd apps/backend
     export PYTHONPATH=$PYTHONPATH:. 
     python3 -m src.workers.jobs.stream_chain
     ```
   - This worker will "detect" a transaction for your customer and send it to the **Anomaly Scorer**.
3. **Verify Alert**:
   - Check **`GET /api/v1/alerts`**. You should see a new alert with a `risk_score` and a natural language `explanation_nl` generated by GPT-4.

## 5. Test Live Audit
Check **`GET /api/v1/audit`** to see the system logs for every alert created and every document polled.
